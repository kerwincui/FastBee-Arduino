# 配置导入导出指南

本文档说明当前 Web 管理界面中“设备配置 > 高级配置 > 配置导入/导出”的使用方式。该功能只处理 LittleFS 中 `/config` 目录下的 JSON 配置文件，不包含 Web 静态资源、日志文件或固件。

## 支持的配置类型

| 类型 | 文件 | 说明 |
| --- | --- | --- |
| 全部配置包 | `fastbee-config-*.json` | 包含多个配置文件的备份包 |
| 设备信息 | `device.json` | 设备 ID、名称、缓存时间、NTP 等基础信息 |
| 网络配置 | `network.json` | STA/AP、静态 IP、mDNS、重连策略 |
| 外设配置 | `peripherals.json` | 本地 GPIO、UART、I2C、传感器、显示屏等外设 |
| 外设执行 | `periph_exec.json` | 本地自动化规则、触发条件、动作列表 |
| 通信协议 | `protocol.json` | MQTT、Modbus RTU、子设备、映射任务 |
| 用户配置 | `users.json` | 完整版多用户配置；精简版通常会跳过 |
| 角色配置 | `roles.json` | 完整版角色权限配置；精简版通常会跳过 |

精简版开启 `FASTBEE_ENABLE_CONFIG_TRANSFER=1`，但关闭多用户、角色、文件管理和 OTA。导入时如果遇到精简版不支持的 `users.json`、`roles.json`，后端会按当前编译开关跳过或拒绝，避免破坏单管理员模式。

## 推荐使用方式

日常备份建议优先按类型导出，尤其是外设执行、外设配置和通信协议。这样恢复时只覆盖目标模块，风险更低，也更容易在不同设备之间迁移。

完整迁移设备时再选择“全部配置包”。导入全部配置包后建议重启设备，使外设初始化、协议启动和规则调度都从新配置干净加载。

## 为什么不再使用大文件内联导入

ESP32 Web 服务解析表单时会临时占用 RAM。较大的配置文件如果通过 `/api/config/transfer/import` 一次性内联上传，容易触发：

```json
{
  "success": false,
  "error": "Config content too large for inline import",
  "code": 413
}
```

当前前端已经改为统一使用 `/api/config/transfer/import-chunk` 分片导入，每片按 UTF-8 字节控制在约 `6KB`。这样即使配置里包含较多中文文案，也不会因为字符数和字节数不一致导致超过后端单片限制。

## Web 操作流程

1. 打开“设备配置”。
2. 切换到“高级配置”。
3. 在“配置类型”中选择要导入/导出的范围。
4. 导出时点击“导出配置包”。如果选择单个类型，会直接下载对应的 `.json` 文件；如果选择全部，会下载 `fastbee-config-*.json` 配置包。
5. 导入时先点击“选择配置文件”，可以选择单个 `.json` 或完整配置包。
6. 点击“导入配置”，确认覆盖后开始分片上传。
7. 导入完成后建议重启设备。

## 文件大小限制

| 限制项 | 当前策略 |
| --- | --- |
| 单个配置文件 | 最大约 `128KB` |
| 导入分片 | 每片约 `6KB` |
| 配置包选择总量 | 前端限制约 `512KB` |
| 后端分片数 | 单文件最多 32 片 |

如果某个配置文件超过 128KB，应优先检查是否存在过多测试规则、历史无效外设或过大的脚本内容。精简版不建议保留大量未启用的规则和历史配置。

## 导入后的生效范围

| 文件 | 是否建议重启 | 原因 |
| --- | :---: | --- |
| `device.json` | 建议 | 设备身份、缓存时间、NTP 等会影响多个页面和服务 |
| `network.json` | 建议 | 网络模式、mDNS、静态 IP 需要重新初始化 |
| `peripherals.json` | 建议 | 外设引脚和驱动初始化更适合冷启动加载 |
| `periph_exec.json` | 建议 | 规则调度器需要重新加载规则和任务 |
| `protocol.json` | 建议 | MQTT/Modbus RTU 启停、串口绑定和轮询任务需要重新初始化 |

## 排错

| 现象 | 建议处理 |
| --- | --- |
| 提示 `413` | 确认前端已更新到当前版本；当前版本应走分片导入 |
| 提示文件名非法 | 仅支持 `/config` 下安全的 `.json` 文件名，不支持路径和 `..` |
| 导入后页面仍显示旧数据 | 清除浏览器缓存或刷新页面，必要时重启设备 |
| 导入后协议未启动 | 保存/导入 `protocol.json` 后重启设备 |
| 导入后传感器联动无效 | 确认 `peripherals.json` 中传感器已启用，并且外设执行中存在传感器采集动作 |
