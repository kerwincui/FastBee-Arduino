# 实验24：SD卡读写

## 实验概述

通过 SPI 或 SDIO 接口读写 SD 卡/TF 卡，用于数据存储、日志记录和配置文件管理。当前 FastBee 已保留 `SDIO` 外设类型（type 37）和配置框架，但运行时 SD 卡挂载、文件读写动作尚未内置。

> **注意**：本实验文档提供接线和配置占位。真正启用 SD 卡读写建议放在 ESP32-S3 full 固件中扩展，避免普通 ESP32 slim 固件增加过多资源占用。

## 硬件接线

### SPI 模式

| 开发板标识 | GPIO引脚 | SD卡引脚 |
|-----------|---------|---------|
| MOSI | GPIO23 | DI（CMD） |
| MISO | GPIO19 | DO（DAT0） |
| CLK | GPIO18 | CLK |
| CS | GPIO5 | CS（DAT3） |

### SDIO 模式（更快）

| 开发板标识 | GPIO引脚 | SD卡引脚 |
|-----------|---------|---------|
| CLK | GPIO14 | CLK |
| CMD | GPIO15 | CMD |
| D0 | GPIO2 | DAT0 |
| D1 | GPIO4 | DAT1 |
| D2 | GPIO12 | DAT2 |
| D3 | GPIO13 | DAT3 |

## FastBee 外设配置

> ⚠️ **注意**：当前SD卡配置框架已就绪，但运行时驱动和文件动作待补充

### 方式1：Web界面配置（推荐）

#### 步骤1：进入外设管理页面

1. 打开浏览器访问 ESP32 IP 地址
2. 登录后点击左侧菜单 **外设配置**

#### 步骤2：添加SD卡外设（SPI模式）

1. 点击 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 填写配置：

   **基本信息（左列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `sd_spi` | SD卡-SPI |
   | **名称** | `SD卡-SPI` | 显示名称 |
   | **外设类型** | 选择 **SD卡** (type: 37) | SDIO/SD卡 |
   | **启用** | ❌ 关闭 | 固件驱动待补充 |

   **引脚和参数（右列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **引脚配置** | `23,19,18,5` | MOSI,MISO,CLK,CS |

3. 点击 **保存**

> 💡 **提示**：暂时保持禁用状态，待固件补充SD驱动后再启用

### SPI 模式

```json

      "id": "sd_spi",
      "name": "SD卡-SPI",
      "type": 37,
      "enabled": false,
      "pins": [23, 19, 18, 5],
      "params": {}
    }
  ]
}
```

## 注意事项

1. **格式**：SD 卡需格式化为 FAT32（<32GB）或 exFAT（≥32GB）
2. **供电**：SD 卡工作电流可达 100mA，确保 3.3V 供电稳定
3. **SPI 共享**：SD 卡使用 SPI 总线时，CS 引脚用于选择设备
4. **GPIO12 冲突**：SDIO 模式下 GPIO12 为 DAT2，启动时不能为高电平
5. **写入频率**：避免过于频繁的写入操作，建议批量写入
6. **文件系统**：FastBee 使用 LittleFS 管理内部 Flash，SD 卡使用 FAT 文件系统
7. **热插拔**：不支持热插拔，SD 卡需在断电时插拔
8. **当前状态**：配置框架就绪，运行时驱动和文件动作待补
