# 实验11：PWM呼吸灯

## 实验概述

通过 PWM（脉宽调制）输出控制 LED 亮度，实现从暗到亮再从亮到暗的呼吸灯效果。PWM 通过调节占空比来模拟模拟电压输出。

## 硬件接线

| 开发板标识 | GPIO引脚 | 连接设备 |
|-----------|---------|---------|
| D1 | GPIO15 | LED（PWM调光） |

> ESP32 拥有 16 个 PWM 通道，几乎所有 GPIO 都可输出 PWM 信号。

本实验的 Web 操作入口如下：先在“外设配置”创建硬件对象，再在“外设执行”添加采集、控制或显示规则。新增外设时建议先保持禁用，确认接线后再启用。

### 方式1：Web界面配置（推荐）

#### 步骤1：进入外设管理页面

1. 打开浏览器访问 ESP32 IP 地址
2. 登录后点击左侧菜单 **外设配置**

#### 步骤2：添加PWM输出外设

1. 点击 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 填写配置：

   **基本信息（左列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `pwm_breath` | PWM呼吸灯 |
   | **名称** | `呼吸灯-PWM` | 显示名称 |
   | **外设类型** | 选择 **PWM输出** (type: 17) | PWM调光 |
   | **启用** | ✅ 开启 | 默认启用 |

   **引脚和参数（右列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **引脚配置** | `15` | D1对应GPIO15 |
   | **PWM通道** | `0` | 0-15，任选一个空闲通道 |
   | **PWM频率** | `5000` | 5kHz（LED调光推荐1kHz以上） |
   | **PWM分辨率** | `8` | 8位=0~255级亮度 |
   | **默认占空比** | `0` | 默认最亮（共阳LED） |

3. 点击 **保存**

#### 步骤3：验证配置

1. 在外设列表中查看新增的PWM外设
2. 确认状态为 **运行中**
3. 点击 **控制** 按钮，输入0-255的占空比测试亮度变化

> 💡 **提示**：共阳LED，duty=0最亮，duty=255最暗

## JSON 配置示例

```json
{
  "peripherals": [
    {
      "id": "pwm_breath",
      "name": "呼吸灯-PWM",
      "type": 17,
      "enabled": false,
      "pins": [15],
      "params": {
        "pwmChannel": 0,
        "pwmFrequency": 5000,
        "pwmResolution": 8,
        "defaultDuty": 0
      }
    }
  ]
}
```

### 参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| pwmChannel | PWM通道号（0-15） | 0 |
| pwmFrequency | 频率（Hz），高频无闪烁 | 5000 |
| pwmResolution | 分辨率（位），8位=0~255 | 8 |
| defaultDuty | 初始占空比 | 0 |

## 外设执行联动

### 场景1：远程调节亮度（平台触发）

**功能**：通过云平台下发指令（0-255）控制LED亮度

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击左侧菜单 **外设配置** → 切换到 **外设执行管理** 标签
2. 点击 **<i class="fas fa-plus"></i> 新增规则** 按钮
3. 填写基础配置：
   - **规则名称**：`远程调节亮度`
   - **上报数据**：✅ 启用
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **平台触发** | 接收云平台指令 |
   | **操作符** | 选择 **设置模式** | 接收数据作为动作参数 |

**步骤3：配置动作**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **设置PWM**
   - **目标外设**：选择 `pwm_breath`

3. 点击 **保存** 按钮

**测试方法**：

通过云平台发送：
- `0` → 最亮（共阳LED）
- `128` → 50%亮度
- `255` → 最暗（熄灭）

### 场景2：自动呼吸效果（定时触发+脚本）

**功能**：使用脚本实现LED自动呼吸效果（渐亮→渐暗循环）

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击 **<i class="fas fa-plus"></i> 新增规则**
2. 填写基础配置：
   - **规则名称**：`LED呼吸灯`
   - **上报数据**：✅ 启用
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **定时触发** | 定时执行脚本 |
   | **定时模式** | 选择 **固定间隔** | 快速刷新 |
   | **间隔时间** | `20` | 20ms（呼吸效果平滑） |

**步骤3：配置动作（脚本）**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **命令脚本**
   - **动作参数**：
   ```
   var d=getVar('breath_duty',0); 
   var dir=getVar('breath_dir',1); 
   d+=dir*5; 
   if(d>=255){d=255;dir=-1;} 
   if(d<=0){d=0;dir=1;} 
   pwmWrite('pwm_breath',d); 
   setVar('breath_duty',d); 
   setVar('breath_dir',dir);
   ```

3. 点击 **保存** 按钮

**脚本说明**：

| 代码 | 说明 |
|------|------|
| `getVar('breath_duty',0)` | 获取当前占空比，默认0 |
| `getVar('breath_dir',1)` | 获取方向，1=渐亮，-1=渐暗 |
| `d+=dir*5` | 每次变化5级 |
| `pwmWrite('pwm_breath',d)` | 设置PWM占空比 |
| `setVar(...)` | 保存变量供下次使用 |

> ⚠️ **注意**：脚本每20ms执行一次，占用系统资源，不建议同时运行多个脚本

## 注意事项

1. **频率选择**：LED 调光建议 1kHz 以上避免闪烁，5kHz 是常用值
2. **分辨率**：8位（256级）足够LED调光，12位（4096级）用于精细控制
3. **通道冲突**：不同引脚可以共享同一 PWM 通道（同频率同占空比）
4. **低电平点亮**：普中开发板LED共阳，duty=0最亮，duty=255最暗
5. **多路PWM**：ESP32 支持同时输出 16 路独立 PWM
