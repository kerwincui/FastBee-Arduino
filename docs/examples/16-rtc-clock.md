# 实验16：RTC时钟（系统NTP）

## 实验概述

ESP32 内部没有独立的 RTC 时钟芯片（断电丢失），但通过 WiFi 连接后可使用 NTP（网络时间协议）自动同步网络时间。FastBee 在联网后自动进行 NTP 时间同步，可用于定时任务和时间显示。

## 硬件接线

本实验无需额外硬件，利用 ESP32 的 WiFi 联网能力获取 NTP 时间。

> 如需断电保持时间，请参考实验17（DS1302外部RTC）。

## FastBee 外设配置

### 系统时间配置（NTP）

NTP 时间同步为系统内置功能，无需额外配置外设。

#### 步骤1：确保WiFi已连接

1. 打开浏览器访问 ESP32 IP 地址
2. 登录后确认网络状态为 **已连接**

#### 步骤2：配置NTP服务器

1. 点击左侧菜单 **系统管理** → **时间设置**
2. 填写配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **NTP服务器** | `ntp.aliyun.com` | 阿里云NTP（默认） |
   | **时区** | `UTC+8` | 中国标准时间 |

3. 点击 **保存** 按钮
4. 系统自动同步网络时间

> 💡 **提示**：NTP同步后，设备即使短暂离线也能保持时间继续计时

## 外设执行联动

### 场景1：每秒更新时间显示（定时触发）

**功能**：每秒更新数码管显示当前系统时间（带冒号闪烁）

#### Web界面配置步骤

**步骤1：确保已配置TM1637数码管**

- 外设ID：`tm1637_clock`
- 类型：数码管显示

**步骤2：创建规则**

1. 点击左侧菜单 **外设配置** → 切换到 **外设执行管理** 标签
2. 点击 **<i class="fas fa-plus"></i> 新增规则** 按钮
3. 填写基础配置：
   - **规则名称**：`时钟显示`
   - **上报数据**：✅ 启用
   - **启用**：✅ 启用

**步骤3：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **定时触发** | 定时更新 |
   | **定时模式** | 选择 **固定间隔** | 按毫秒间隔 |
   | **间隔时间** | `1000` | 1秒更新一次 |

**步骤4：配置动作**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **显示时间**
   - **目标外设**：选择 `tm1637_clock`
   - **显示冒号**：✅ 启用

3. 点击 **保存** 按钮

### 场景2：每天6点自动开灯（Cron定时）

**功能**：每天早上6点自动开启LED灯

#### Web界面配置步骤

1. 创建规则，名称：`每天6点开灯`
2. 触发器配置：
   - **触发类型**：选择 **定时触发**
   - **定时模式**：选择 **Cron表达式**
   - **Cron表达式**：`0 6 * * *`（每天6:00）
3. 动作配置：
   - **动作类型**：选择 **低电平**（共阳LED）
   - **目标外设**：选择 `led_d1`
4. 点击 **保存**

### Cron表达式示例

| Cron表达式 | 说明 | 执行时间 |
|-----------|------|---------|
| `0 6 * * *` | 每天6:00 | 每天早上6点 |
| `0 18 * * *` | 每天18:00 | 每天傍晚6点 |
| `*/5 * * * *` | 每5分钟 | 每隔5分钟 |
| `0 */2 * * *` | 每2小时 | 每隔2小时整点 |

## 注意事项

1. **NTP 依赖网络**：首次启动需联网获取时间，离线模式下时间从上次同步继续计时
2. **精度**：NTP 同步精度约 ±50ms，本地晶振漂移约 ±2ppm
3. **断电丢失**：ESP32 断电后 RTC 时间清零，重新上电需再次 NTP 同步
4. **Cron 表达式**：支持标准5字段格式：分 时 日 月 周
5. **时区**：系统默认 UTC+8，可在设备配置中修改
