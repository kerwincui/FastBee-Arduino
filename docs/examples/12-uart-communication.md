# 实验12：UART串口通信

## 实验概述

通过 UART（通用异步收发器）实现 ESP32 与外部设备的串口通信。ESP32 拥有 3 个 UART 端口，FastBee 支持配置串口用于调试输出、Modbus 通信或自定义数据交换。

## 硬件接线

| UART端口 | TX引脚 | RX引脚 | 用途 |
|----------|--------|--------|------|
| UART0 | GPIO1 | GPIO3 | USB 调试输出 |
| UART1 | GPIO16 | GPIO17 | Modbus/外部设备 |
| UART2 | GPIO18 | GPIO19 | 扩展串口 |

> UART0 默认用于 USB 调试输出，UART1/2 可用于连接外部设备。

## FastBee 外设配置

本实验的 Web 操作入口如下：先在“外设配置”创建硬件对象，再在“外设执行”添加采集、控制或显示规则。新增外设时建议先保持禁用，确认接线后再启用。

### 方式1：Web界面配置（推荐）

#### 步骤1：进入外设管理页面

1. 打开浏览器访问 ESP32 IP 地址
2. 登录后点击左侧菜单 **外设配置**

#### 步骤2：添加UART串口外设

1. 点击 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 填写配置：

   **基本信息（左列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `uart1` | 串口1 |
   | **名称** | `串口1-Modbus` | 显示名称 |
   | **外设类型** | 选择 **串口通信** (type: 1) | UART通信 |
   | **启用** | ✅ 开启 | 默认启用 |

   **引脚和参数（右列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **引脚配置** | `16,17` | TX=GPIO16, RX=GPIO17 |
   | **波特率** | `9600` | 常用9600或115200 |
   | **数据位** | `8` | 7或8 |
   | **停止位** | `1` | 1或2 |
   | **校验位** | **无** | 无/奇/偶 |

3. 点击 **保存**

> ⚠️ **注意**：UART0（GPIO1/3）用于USB调试，不建议重新分配

#### 步骤3：验证配置

1. 在外设列表中查看新增的UART外设
2. 确认状态为 **运行中**

## JSON 配置示例

```json
{
  "peripherals": [
    {
      "id": "uart0",
      "name": "串口0-调试",
      "type": 1,
      "enabled": false,
      "pins": [1, 3],
      "params": {
        "baudRate": 115200,
        "dataBits": 8,
        "stopBits": 1,
        "parity": 0
      }
    },
    {
      "id": "uart1",
      "name": "串口1-Modbus",
      "type": 1,
      "enabled": false,
      "pins": [16, 17],
      "params": {
        "baudRate": 9600,
        "dataBits": 8,
        "stopBits": 1,
        "parity": 0
      }
    }
  ]
}
```

### 参数说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| baudRate | 波特率 | 9600, 19200, 38400, 57600, 115200 |
| dataBits | 数据位 | 7, 8 |
| stopBits | 停止位 | 1, 2 |
| parity | 校验位 | 0=无, 1=奇, 2=偶 |

## 外设执行联动

### 场景：串口数据发送（平台触发）

**功能**：通过云平台下发指令，通过UART1发送数据到外部设备

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击左侧菜单 **外设配置** → 切换到 **外设执行管理** 标签
2. 点击 **<i class="fas fa-plus"></i> 新增规则** 按钮
3. 填写基础配置：
   - **规则名称**：`串口发送数据`
   - **上报数据**：✅ 启用
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **平台触发** | 接收云平台指令 |
   | **操作符** | 选择 **设置模式** | 接收数据作为动作参数 |

**步骤3：配置动作**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **串口发送**
   - **目标外设**：选择 `uart1`

3. 点击 **保存** 按钮

**测试方法**：

通过云平台发送任意字符串，例如：`Hello UART`
数据将通过UART1的TX引脚（GPIO16）发送出去

### UART参数说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| 波特率 | 通信速率 | 9600, 19200, 38400, 57600, 115200 |
| 数据位 | 数据位数 | 7, 8 |
| 停止位 | 停止位数 | 1, 2 |
| 校验位 | 校验方式 | 无, 奇校验, 偶校验 |

## 注意事项

1. **UART0 占用**：UART0（GPIO1/3）用于 USB 调试，不建议重新分配
2. **引脚映射**：ESP32 的 UART TX/RX 可映射到任意 GPIO（除 GPIO6-11）
3. **电压匹配**：ESP32 为 3.3V 电平，连接 5V 设备需电平转换
4. **Modbus 配置**：用于 Modbus 时波特率通常为 9600，需与从站设备匹配
5. **缓冲区**：UART 接收缓冲区默认 256 字节，大数据量需及时读取
6. **ESP32-S3**：S3 的 UART0 通过 USB CDC 输出，物理 UART 从 UART1 开始
