# 实验4：蜂鸣器控制

## 实验概述

通过 GPIO 输出或 PWM 信号驱动蜂鸣器发声。无源蜂鸣器需要 PWM 方波驱动（可调频率），有源蜂鸣器仅需高/低电平开关控制。

## 硬件接线

| 开发板标识 | GPIO引脚 | 连接设备 |
|-----------|---------|---------|
| BEEP | GPIO4 | 有源/无源蜂鸣器 |

> 普中开发板蜂鸣器接在 GPIO4，有源蜂鸣器高电平驱动。

## FastBee 外设配置

本实验的 Web 操作入口如下：先在“外设配置”创建硬件对象，再在“外设执行”添加采集、控制或显示规则。新增外设时建议先保持禁用，确认接线后再启用。

### 方式1：Web界面配置（推荐）

#### 方案A：有源蜂鸣器（GPIO开关控制）

**步骤1：进入外设管理页面**

1. 打开浏览器访问 ESP32 IP 地址
2. 登录后点击左侧菜单 **外设配置**

**步骤2：添加蜂鸣器外设**

1. 点击 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 填写配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `buzzer_gpio` | 留空自动生成 |
   | **名称** | `蜂鸣器-开关` | 显示名称 |
   | **外设类型** | **数字输出** (type: 12) | GPIO输出 |
   | **引脚配置** | `4` | BEEP对应GPIO4 |
   | **初始状态** | **低电平** | 低电平=关闭 |

3. 点击 **保存**

#### 方案B：无源蜂鸣器（PWM频率控制）

**步骤2：添加PWM蜂鸣器外设**

1. 点击 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 填写配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `buzzer_pwm` | 留空自动生成 |
   | **名称** | `蜂鸣器-PWM` | 显示名称 |
   | **外设类型** | **PWM输出** (type: 17) | PWM输出 |
   | **引脚配置** | `4` | BEEP对应GPIO4 |
   | **初始状态** | **低电平** | 默认关闭 |
   | **PWM频率** | `2000` | 2000Hz（2kHz） |
   | **PWM分辨率** | `8` | 8位（0-255） |
   | **默认占空比** | `0` | 默认关闭 |

3. 点击 **保存**

> ⚠️ **注意**：同一引脚只能配置一种模式，GPIO和PWM选其一

#### 步骤3：验证配置

1. 在外设列表中查看新增的蜂鸣器
2. 点击 **控制** 按钮测试发声

## 外设执行联动

### 场景1：定时报警（定时触发）

**功能**：每5秒响一次，持续200ms

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击左侧菜单 **外设配置** → 切换到 **外设执行管理** 标签
2. 点击 **<i class="fas fa-plus"></i> 新增规则** 按钮
3. 填写基础配置：
   - **规则名称**：`蜂鸣器定时报警`
   - **上报数据**：❌ 禁用
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **定时触发** | 按时间间隔执行 |
   | **定时模式** | 选择 **间隔定时** | 每隔X秒执行 |
   | **间隔秒数** | `5` | 每5秒执行一次 |

**步骤3：配置动作（需要3个动作）**

**动作1：启动蜂鸣器**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **高电平**
   - **目标外设**：选择 `buzzer_gpio`

**动作2：延迟200ms**

1. 再次点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **命令脚本**
   - **脚本内容**：`DELAY 200`

**动作3：关闭蜂鸣器**

1. 第三次点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **低电平**
   - **目标外设**：选择 `buzzer_gpio`

3. 点击 **保存** 按钮

**执行流程**：
```
每5秒
  ↓
启动蜂鸣器（HIGH）
  ↓
延迟200ms
  ↓
关闭蜂鸣器（LOW）
```

### 场景2：PWM音调控制（平台触发）

**功能**：通过云平台下发指令控制音调频率

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击 **<i class="fas fa-plus"></i> 新增规则**
2. 填写基础配置：
   - **规则名称**：`蜂鸣器播放音调`
   - **上报数据**：✅ 启用
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **平台触发** | 接收云平台指令 |
   | **操作符** | 选择 **匹配模式** | 接收数据等于比较值时触发 |
   | **比较值** | `{"freq":1000}` | 匹配的指令 |

**步骤3：配置动作**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **设置PWM**
   - **目标外设**：选择 `buzzer_pwm`
   - **动作参数**：`{"duty":128,"frequency":1000}`

3. 点击 **保存** 按钮

### 常用音调频率

| 音符 | 频率(Hz) | 说明 |
|------|---------|------|
| DO | 523 | 中央C |
| RE | 587 | D |
| MI | 659 | E |
| FA | 698 | F |
| SO | 784 | G |
| LA | 880 | A |
| SI | 988 | B |
| DO+ | 1046 | 高音C |

## 注意事项

1. **有源 vs 无源**：有源蜂鸣器内置振荡器，给电就响；无源需要 PWM 方波
2. **PWM频率范围**：人耳可听范围 20Hz~20kHz，常用报警音 1kHz~4kHz
3. **占空比**：无源蜂鸣器建议 50%（duty=128/8bit），过低可能不响
4. **噪音控制**：调试时可降低占空比减小音量
