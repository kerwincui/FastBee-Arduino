# 实验1：点亮第一个LED

## 实验概述

通过 GPIO 数字输出控制 LED 灯的亮灭，是嵌入式开发的入门实验。普中开发板上有4个 LED（D1-D4），分别连接到 GPIO15、GPIO2、GPIO0、GPIO4。

## 硬件接线

| 开发板标识 | GPIO引脚 | 连接设备 |
|-----------|---------|---------|
| D1 | GPIO15 | LED（低电平点亮） |
| D2 | GPIO2 | LED（低电平点亮） |
| D3 | GPIO0 | LED（低电平点亮） |
| D4 | GPIO4 | LED（低电平点亮） |

> 普中开发板 LED 为共阳接法，GPIO 输出低电平时 LED 点亮。

## FastBee 外设配置

本实验的 Web 操作入口如下：先在“外设配置”创建硬件对象，再在“外设执行”添加采集、控制或显示规则。新增外设时建议先保持禁用，确认接线后再启用。

### 方式1：Web界面配置（推荐）

#### 步骤1：进入外设管理页面

1. 打开浏览器，访问 ESP32 的 IP 地址（如 `http://192.168.1.100`）
2. 输入用户名和密码登录
3. 点击左侧菜单 **外设配置**

#### 步骤2：添加LED外设

1. 点击右上角 **<i class="fas fa-plus"></i> 新增外设** 按钮
2. 在弹出的模态窗中填写：

   **基本信息（左列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **外设ID** | `led_d1` | 留空自动生成，建议手动填写有意义的ID |
   | **名称** | `LED-D1` | 显示名称，建议使用开发板标识 |
   | **外设类型** | 选择 **数字输出** (type: 12) | 从下拉框选择 |
   | **启用** | ✅ 开启 | 默认启用 |

   **引脚和参数（右列）：**
   
   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **引脚配置** | `15` | 填写GPIO编号，D1对应GPIO15 |
   | **初始状态** | 选择 **低电平** | 普中开发板LED为低电平点亮 |

3. 点击 **保存** 按钮
4. 系统验证后写入配置，列表自动刷新

#### 步骤3：重复添加其他LED

按照步骤2，继续添加D2、D3、D4：

| 外设ID | 名称 | 引脚 | 初始状态 |
|--------|------|------|---------|
| `led_d2` | LED-D2 | `2` | 低电平 |
| `led_d3` | LED-D3 | `0` | 低电平 |
| `led_d4` | LED-D4 | `4` | 低电平 |

> ⚠️ **注意**：GPIO0在烧录固件时需要拉低，正常运行时可作为LED输出

#### 步骤4：验证配置

1. 在外设列表中查看新增的4个LED外设
2. 确认 **状态** 列显示为 **运行中**（绿色）
3. 点击每个外设的 **控制** 按钮，测试LED亮灭

## 外设执行联动

### 场景1：远程控制LED（平台触发）

**功能**：通过云平台下发的MQTT/HTTP指令控制LED

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击左侧菜单 **外设配置** → 切换到 **外设执行管理** 标签
2. 点击右上角 **<i class="fas fa-plus"></i> 新增规则** 按钮
3. 填写基础配置：
   - **规则名称**：`远程控制LED-D1`
   - **上报数据**：✅ 启用（执行后上报状态到云平台）
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **平台触发** | 接收云平台指令 |
   | **目标外设** | 留空或选择 `led_d1` | 接收指令的外设 |
   | **操作符** | 选择 **匹配模式** | 接收数据等于比较值时触发 |
   | **比较值** | `{"cmd":"on"}` | 匹配的指令内容 |

**步骤3：配置动作**

1. 点击 **添加动作** 按钮
2. 填写动作配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **动作类型** | 选择 **低电平** | GPIO输出LOW，点亮LED |
   | **目标外设** | 选择 `led_d1` | 控制的外设ID |
   | **执行模式** | 选择 **同步** | 等待动作完成 |

3. 点击 **保存** 按钮

**测试方法**：

通过云平台发送MQTT消息：
```json
{"cmd":"on"}
```

### 场景2：定时闪烁LED（定时触发）

**功能**：让LED每秒闪烁一次

#### Web界面配置步骤

**步骤1：创建规则**

1. 点击 **<i class="fas fa-plus"></i> 新增规则**
2. 填写基础配置：
   - **规则名称**：`LED-D1闪烁`
   - **上报数据**：❌ 禁用（不需要上报）
   - **启用**：✅ 启用

**步骤2：配置触发器**

1. 点击 **添加触发** 按钮
2. 填写触发器配置：

   | 字段 | 填写内容 | 说明 |
   |------|---------|------|
   | **触发类型** | 选择 **定时触发** | 按时间间隔执行 |
   | **定时模式** | 选择 **间隔定时** | 每隔X秒执行 |
   | **间隔秒数** | `1` | 每1秒执行一次 |

**步骤3：配置动作（需要2个动作实现闪烁）**

**动作1：点亮LED**

1. 点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **低电平**
   - **目标外设**：选择 `led_d1`

**动作2：延迟后熄灭LED**

1. 再次点击 **添加动作** 按钮
2. 填写：
   - **动作类型**：选择 **命令脚本**
   - **脚本内容**：
     ```
     DELAY 500
     PERIPH led_d1 HIGH
     ```
   - **执行模式**：选择 **异步**

3. 点击 **保存** 按钮

**执行流程**：
```
每隔1秒
  ↓
点亮LED（低电平）
  ↓
延迟500ms
  ↓
熄灭LED（高电平）
  ↓
等待下一个1秒周期
```

```json
{
  "name": "远程控制LED-D1",
  "enabled": true,
  "triggers": [
    { "type": "platform", "params": { "topic": "led/d1" } }
  ],
  "actions": [
    { "type": "gpio_write", "params": { "peripheralId": "led_d1", "value": 0 } }
  ]
}
```

## 注意事项

1. **GPIO0**：烧录时需拉低，正常运行时可作为 LED 输出，但调试时需注意
2. **GPIO2**：部分 ESP32 模块上内置 LED 也连接到此引脚
3. **电平逻辑**：普中开发板 LED 为低电平点亮，与常见的高电平点亮相反
4. **电流限制**：ESP32 GPIO 输出最大 12mA，板载 LED 已有限流电阻
