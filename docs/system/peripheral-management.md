# 外设配置完整指南

## 一、功能概述

外设管理页面是 FastBee 系统的**硬件抽象层配置中心**，通过 Web 界面将物理硬件（GPIO引脚、传感器、通信接口等）映射为系统可识别的**虚拟外设对象**。所有配置存储在 `/config/peripherals.json` 文件中。

### 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| **外设ID** | 唯一标识符，用于规则引用 | `led1`, `dht_temp`, `relay_pump` |
| **外设类型** | 硬件类型编码（type值） | 12=数字输出, 38=传感器 |
| **引脚配置** | 物理GPIO引脚编号数组 | `[15]`, `[4, 5]`（I2C的SDA,SCL） |
| **参数(params)** | 类型特定的配置项 | `{initialState: 1}`, `{baudrate: 115200}` |
| **启用状态** | 控制外设是否初始化 | `enabled: true` 时占用硬件资源 |

## 二、Web界面操作指南

### 2.1 添加外设

#### 操作步骤

1. **进入外设管理页面**
   - 点击左侧菜单 **外设配置**
   - 页面显示现有外设列表（表格形式）

2. **点击新增按钮**
   - 点击右上角 **<i class="fas fa-plus"></i> 新增外设** 按钮
   - 弹出外设配置模态窗

3. **填写基本信息**（左列）

   | 字段 | 必填 | 说明 | 示例 |
   |------|------|------|------|
   | **外设ID** | 否 | 留空自动生成（格式：`periph_<timestamp>`） | `led_room` |
   | **名称** | ✅ | 显示名称，建议有业务含义 | `客厅主灯` |
   | **外设类型** | ✅ | 从下拉框选择硬件类型 | `数字输出` |
   | **启用** | - | 开关控制，默认启用 | ✅ 开启 |

4. **配置引脚和参数**（右列）

   - **引脚配置**：输入GPIO编号，多个引脚用逗号分隔
   - **动态参数区**：根据类型自动显示对应参数表单

5. **保存配置**
   - 点击 **保存** 按钮
   - 系统验证后写入 `peripherals.json`
   - 列表自动刷新显示新外设

#### 界面截图说明

```
┌─────────────────────────────────────────────┐
│  新增外设                              [×]  │
├─────────────────────────────────────────────┤
│  【左列：基本配置】    【右列：引脚和参数】   │
│  外设ID: [led_1____]  引脚配置: [2,4,5___]  │
│  名称:   [客厅灯____]  ┌─────────────────┐ │
│  类型:   [数字输出▼]  │ GPIO参数        │ │
│  启用:   [●══○____]  │ 初始状态: [低▼] │ │
│                       └─────────────────┘ │
├─────────────────────────────────────────────┤
│               [取消]    [保存]              │
└─────────────────────────────────────────────┘
```

### 2.2 编辑外设

#### 操作步骤

1. 在外设列表中找到目标外设
2. 点击该行的 **编辑** 按钮（蓝色）
3. 模态窗自动填充当前配置
4. 修改需要调整的字段
5. 点击 **保存** 确认更改

#### 注意事项

- ⚠️ **外设ID不可修改**：编辑时ID字段为只读
- ⚠️ **引脚变更会重新初始化**：修改引脚后外设会重新初始化
- ⚠️ **引用检查**：如果有规则引用该外设，修改ID会导致规则失效

---

### 2.3 启用/禁用外设

#### 操作方式

| 方式 | 操作 | 效果 |
|------|------|------|
| **列表快捷操作** | 点击 **启用/禁用** 按钮 | 立即切换状态 |
| **编辑模态窗** | 切换 **启用** 开关后保存 | 保存后生效 |

#### 状态说明

| 状态值 | 状态名称 | 含义 | 资源占用 |
|--------|---------|------|---------|
| 0 | 禁用 | 外设未初始化 | ❌ 不占用GPIO |
| 1 | 启用 | 已初始化待运行 | ✅ 占用引脚 |
| 2 | 已初始化 | 驱动加载完成 | ✅ 占用引脚 |
| 3 | 运行中 | 正常工作 | ✅ 占用引脚 |
| 4 | 错误 | 初始化失败 | ⚠️ 可能占用 |

#### 使用场景

- **临时停用**：禁用而不删除，保留配置
- **调试隔离**：禁用冲突的外设排查问题
- **节省资源**：禁用未使用的外设释放GPIO

---

### 2.4 删除外设

#### 操作步骤

1. 点击该行的 **删除** 按钮（红色）
2. 系统弹出确认对话框
3. 确认后从 `peripherals.json` 移除

#### ⚠️ 重要警告

- ❌ **删除不可恢复**：配置永久丢失（除非有备份）
- 🔗 **检查引用**：删除前确认没有规则引用该外设
- 💡 **建议先禁用**：不确定时先禁用观察几天

---

### 2.5 筛选和分页

#### 类型筛选

下拉框选项：
- **全部外设**：显示所有类型
- **通信接口**：UART, I2C, SPI, CAN, USB
- **GPIO接口**：数字输入/输出, PWM, 中断, 触摸
- **模拟信号**：ADC, DAC
- **调试接口**：JTAG, SWD
- **专用外设**：LCD, 传感器, 舵机, 步进电机等

#### 分页控制

- 每页显示 **10条** 记录
- 底部显示分页导航（上一页/页码/下一页）
- 总数超过10条时自动分页

## 三、外设类型详解

### 3.1 GPIO数字输出 (type=12)

#### 适用场景

- LED指示灯
- 继电器控制
- 蜂鸣器
- 直流电机开关

#### 引脚配置

```
单引脚：15
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **initialState** | 整数 | 0 | 0 或 1 | 初始电平状态 |

- `0` = 低电平（LED灭/继电器断开）
- `1` = 高电平（LED亮/继电器闭合）

#### 配置示例

```json
{
  "id": "led_main",
  "name": "主LED",
  "type": 12,
  "enabled": true,
  "pins": [15],
  "params": {
    "initialState": 0
  }
}
```

#### 接线注意

- **低电平点亮**：LED负极接GPIO，正极串联电阻接VCC
- **高电平点亮**：LED正极接GPIO，负极串联电阻接GND
- **电阻计算**：`R = (VCC - Vf) / If`（红色LED约220Ω）

---

### 3.2 GPIO数字输入 (type=11/13/14)

#### 类型对比

| type值 | 类型名称 | 内部电阻 | 空闲状态 | 适用场景 |
|--------|---------|---------|---------|---------|
| 11 | 数字输入 | 无 | 浮空（不确定） | 有外部上拉/下拉的传感器 |
| 13 | 数字输入(上拉) | 内部上拉 | 高电平(1) | 按键（按下接地） |
| 14 | 数字输入(下拉) | 内部下拉 | 低电平(0) | 按键（按下接VCC） |

#### 引脚配置

```
单引脚：0
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| **debounceMs** | 整数 | 50 | 消抖时间（毫秒），防止按键抖动误触发 |

#### 配置示例（按键）

```json
{
  "id": "btn_power",
  "name": "电源按键",
  "type": 13,
  "enabled": true,
  "pins": [0],
  "params": {
    "debounceMs": 50
  }
}
```

#### 接线注意

- ⚠️ **GPIO0限制**：启动时GPIO0为高电平，拉低会进入下载模式
- **按键接线**：一端接GPIO，一端接GND（上拉模式）
- **消抖处理**：机械按键必须设置debounceMs，建议30-100ms

---

### 3.3 PWM输出 (type=17)

#### 适用场景

- LED呼吸灯
- 电机调速
- 舵机控制（专用舵机类型更方便）
- 蜂鸣器音调

#### 引脚配置

```
单引脚：4
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **initialState** | 整数 | 0 | 0 或 1 | 初始电平 |
| **pwmFreq** | 整数 | 1000 | 1~40000000 | PWM频率(Hz) |
| **pwmResolution** | 整数 | 8 | 1~16 | 分辨率（位） |
| **defaultDuty** | 整数 | 0 | 0~2^resolution-1 | 默认占空比 |

#### 分辨率与占空比关系

| 分辨率 | 最大值 | 50%占空比 | 适用场景 |
|--------|--------|----------|---------|
| 8位 | 255 | 127 | LED调光、简单电机 |
| 10位 | 1023 | 511 | 精确调光 |
| 12位 | 4095 | 2047 | 高精度控制 |
| 16位 | 65535 | 32767 | 音频PWM |

#### 配置示例（呼吸灯）

```json
{
  "id": "pwm_led",
  "name": "呼吸灯",
  "type": 17,
  "enabled": true,
  "pins": [4],
  "params": {
    "initialState": 0,
    "pwmFreq": 1000,
    "pwmResolution": 8,
    "defaultDuty": 0
  }
}
```

#### 频率选择建议

| 场景 | 推荐频率 | 原因 |
|------|---------|------|
| LED调光 | 1000 Hz | 人眼不闪烁 |
| 电机调速 | 20000 Hz | 超出听觉范围，无噪音 |
| 舵机 | 50 Hz | 标准舵机协议 |
| 蜂鸣器 | 1000~4000 Hz | 可听音调 |

---

### 3.4 UART串口 (type=1)

#### 适用场景

- Modbus RTU通信
- 串口调试
- 4G模块（EC801E）
- GPS模块

#### 引脚配置

```
双引脚：39,40
顺序：引脚1=RX(接收), 引脚2=TX(发送)
```

#### ⚠️ 引脚顺序非常重要

```
ESP32引脚 39 → 外部设备的TX（接收外部数据）
ESP32引脚 40 → 外部设备的RX（发送数据到外部）
```

#### 参数说明

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| **baudrate** | 整数 | 115200 | 1200~921600 | 波特率 |
| **dataBits** | 整数 | 8 | 5,6,7,8 | 数据位 |
| **stopBits** | 整数 | 1 | 1,2 | 停止位 |
| **parity** | 整数 | 0 | 0=无,1=奇,2=偶 | 校验位 |

#### 常用波特率

| 波特率 | 适用场景 |
|--------|---------|
| 9600 | GPS模块、老设备 |
| 19200 | 部分传感器 |
| 115200 | **最常用**，4G模块、调试 |
| 921600 | 高速数据传输 |

#### 配置示例

```json
{
  "id": "uart_modbus",
  "name": "Modbus串口",
  "type": 1,
  "enabled": true,
  "pins": [39, 40],
  "params": {
    "baudrate": 9600,
    "dataBits": 8,
    "stopBits": 1,
    "parity": 0
  }
}
```

#### 接线注意

- **交叉连接**：TX接RX，RX接TX
- **共地**：必须连接GND
- **电平匹配**：ESP32为3.3V，5V设备需电平转换

---

### 3.5 传感器 (type=38)

#### 支持类型

| 传感器 | dataField | 单位 | 接线 |
|--------|-----------|------|------|
| DHT11/DHT22 | temperature, humidity | ℃, % | 单GPIO数据引脚 |
| DS18B20 | temperature | ℃ | 单GPIO（单总线） |
| HC-SR04 | distance | cm | 双GPIO（TRIG, ECHO） |
| SHT31/SHT30 | temperature, humidity | ℃, % | I2C（需启用I2C驱动） |
| AHT20/AHT21 | temperature, humidity | ℃, % | I2C |
| BH1750 | light | lux | I2C |
| BMP280 | temperature, pressure | ℃, hPa | I2C（S3-full） |
| MPU6050 | temperature, gyro_x/y/z, accel_x/y/z | ℃, °/s, m/s² | I2C（S3-full） |

#### 引脚配置

```
DHT/DS18B20：单引脚 [21]
HC-SR04：双引脚 [5, 18]（TRIG, ECHO）
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| **sensorType** | 字符串 | "dht" | 传感器类型标识 |
| **dataField** | 字符串 | "temperature" | 主要数据字段 |

#### 配置示例（DHT22）

```json
{
  "id": "dht_living",
  "name": "客厅温湿度",
  "type": 38,
  "enabled": true,
  "pins": [21],
  "params": {
    "sensorType": "dht",
    "dataField": "temperature"
  }
}
```

#### 读取数据

外设执行规则中使用**传感器读取动作**（actionType=19）：

```json
{
  "type": 19,
  "actionType": 19,
  "targetPeriphId": "dht_living",
  "actionValue": "{\"dataField\":\"temperature\"}"
}
```

---

### 3.6 舵机 (type=41)

#### 适用场景

- 云台控制
- 机械臂关节
- 阀门开关
- 摄像头转向

#### 引脚配置

```
单引脚：13
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **minPulseUs** | 整数 | 500 | 200~1000 | 0°脉宽（微秒） |
| **maxPulseUs** | 整数 | 2400 | 2000~3000 | 180°脉宽（微秒） |
| **defaultAngle** | 整数 | 90 | 0~180 | 默认角度 |

#### 配置示例

```json
{
  "id": "servo_camera",
  "name": "摄像头舵机",
  "type": 41,
  "enabled": true,
  "pins": [13],
  "params": {
    "minPulseUs": 500,
    "maxPulseUs": 2400,
    "defaultAngle": 90
  }
}
```

#### 接线注意

- **颜色定义**：红=VCC(5V)，棕/黑=GND，橙/黄=信号
- **供电**：多个舵机需外部5V电源
- **信号电平**：ESP32输出3.3V，大部分舵机可识别

---

### 3.7 步进电机 (type=42)

#### 适用场景

- 3D打印机
- CNC机床
- 精确位置控制
- 自动窗帘

#### 引脚配置

```
四引脚：11,10,9,13
顺序：IN1, IN2, IN3, IN4
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **stepsPerRev** | 整数 | 2048 | 1~4096 | 每圈步数 |
| **speed** | 整数 | 8 | 1~30 | 默认转速(RPM) |

#### 常见电机参数

| 电机型号 | 每圈步数 | 减速比 | 推荐RPM |
|---------|---------|--------|---------|
| 28BYJ-48 | 2048（半步） | 1:64 | 5~15 |
| NEMA 17 | 200（整步） | 1:1 | 10~30 |

#### 配置示例

```json
{
  "id": "stepper_curtain",
  "name": "窗帘步进电机",
  "type": 42,
  "enabled": true,
  "pins": [11, 10, 9, 13],
  "params": {
    "stepsPerRev": 2048,
    "speed": 10
  }
}
```

#### 接线注意

- **驱动板**：必须使用ULN2003等驱动板，不能直连
- **供电**：5V外接电源，ESP32 GPIO仅提供控制信号
- **顺序**：IN1→IN4对应驱动板IN1→IN4

---

### 3.8 WS2812B灯珠 (type=45)

#### 适用场景

- RGB氛围灯
- 状态指示灯带
- 音乐律动灯
- 像素屏

#### 引脚配置

```
单引脚：4
仅需数据引脚(DIN)
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **ledCount** | 整数 | 1 | 1~64 | 灯珠数量 |
| **brightness** | 整数 | 64 | 0~255 | 默认亮度 |

#### ⚠️ 性能限制

| ESP32型号 | 推荐最大数量 | 原因 |
|-----------|------------|------|
| ESP32 slim | 16颗 | 内存限制 |
| ESP32-S3 full | 64颗 | PSRAM支持 |

#### 配置示例

```json
{
  "id": "neopixel_strip",
  "name": "RGB灯带",
  "type": 45,
  "enabled": true,
  "pins": [4],
  "params": {
    "ledCount": 8,
    "brightness": 64
  }
}
```

#### 接线注意

- **方向**：DIN→DOUT串联，箭头方向为数据流向
- **供电**：每颗灯珠60mA（全亮白色），需计算总电流
- **电容**：电源端加1000μF电容稳压
- **电阻**：DIN串联330Ω电阻减少信号反射

---

### 3.9 I2C总线 (type=2)

#### 适用场景

- OLED显示屏
- 传感器（SHT31、AHT20、BH1750、BMP280、MPU6050）
- RTC时钟模块（DS3231）
- EEPROM存储

#### 引脚配置

```
双引脚：21,22
顺序：引脚1=SDA(数据), 引脚2=SCL(时钟)
```

#### 参数说明

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| **frequency** | 整数 | 100000 | 100000/400000 | I2C频率(Hz) |
| **address** | 整数 | 0 | 0~127 | 从机地址（0=主机模式） |

#### 频率选择

| 频率 | 模式 | 适用场景 |
|------|------|---------||
| 100kHz | 标准模式 | 大部分I2C设备，稳定可靠 |
| 400kHz | 快速模式 | 高速传感器，需设备支持 |

#### 配置示例（主机模式）

```json
{
  "id": "i2c_bus",
  "name": "I2C总线",
  "type": 2,
  "enabled": true,
  "pins": [21, 22],
  "params": {
    "frequency": 100000,
    "address": 0
  }
}
```

#### 接线注意

- **上拉电阻**：I2C需要4.7kΩ上拉电阻（大部分模块已内置）
- **总线拓扑**：所有设备并联在SDA和SCL上
- **地址冲突**：每个设备必须有唯一地址
- **线长限制**：建议<1米，长距离需降低频率

---

### 3.10 SPI总线 (type=3)

#### 适用场景

- SD卡模块
- TFT显示屏
- FLASH存储器
- 高速ADC/DAC

#### 引脚配置

```
四引脚：18,23,19,5
顺序：SCK, MOSI, MISO, CS
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **frequency** | 整数 | 1000000 | 1000~80000000 | SPI频率(Hz) |
| **mode** | 整数 | 0 | 0~3 | SPI模式(CPOL,CPHA) |

#### SPI模式详解

| 模式 | CPOL | CPHA | 说明 |
|------|------|------|------|
| 0 | 0 | 0 | **最常用**，空闲低，上升沿采样 |
| 1 | 0 | 1 | 空闲低，下降沿采样 |
| 2 | 1 | 0 | 空闲高，下降沿采样 |
| 3 | 1 | 1 | 空闲高，上升沿采样 |

#### 配置示例

```json
{
  "id": "spi_sd",
  "name": "SD卡SPI",
  "type": 3,
  "enabled": true,
  "pins": [18, 23, 19, 5],
  "params": {
    "frequency": 4000000,
    "mode": 0
  }
}
```

#### 频率选择建议

| 设备 | 推荐频率 | 原因 |
|------|---------|------|
| SD卡 | 4~25 MHz | 标准SD卡支持 |
| TFT屏 | 10~40 MHz | 刷新速度要求 |
| FLASH | 10~50 MHz | 高速读写 |

### 3.11 ADC模拟输入 (type=15/26)

#### 适用场景

- 电压检测
- 电位器读取
- 光敏电阻
- 温度传感器（模拟输出）

#### 引脚配置

```
单引脚：34
```

#### ⚠️ ESP32 ADC限制

| ADC单元 | 可用引脚 | 注意 |
|---------|---------|------|
| ADC1 | 32,33,34,35,36,39 | **推荐使用** |
| ADC2 | 0,2,4,12,13,14,15,25,26,27 | WiFi启用时不可用 |

#### 参数说明

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| **resolution** | 整数 | 12 | 9/10/11/12 | 分辨率（位） |
| **attenuation** | 整数 | 3 | 0/1/2/3 | 衰减系数 |

#### 分辨率与量程

| 分辨率 | 最大值 | 精度 | 适用场景 |
|--------|--------|------|---------|
| 9位 | 511 | ±6.4mV | 快速采样 |
| 10位 | 1023 | ±3.2mV | 一般精度 |
| 11位 | 2047 | ±1.6mV | 较高精度 |
| 12位 | 4095 | ±0.8mV | **最常用** |

#### 衰减系数与电压范围

| 衰减 | 电压范围 | 适用场景 |
|------|---------|---------|
| 0dB | 0~1.1V | 高精度小信号 |
| 2.5dB | 0~1.5V | 传感器输出 |
| 6dB | 0~2.2V | 电池检测 |
| 11dB | 0~3.3V | **最常用**，全量程 |

#### 配置示例

```json
{
  "id": "adc_pot",
  "name": "电位器ADC",
  "type": 26,
  "enabled": true,
  "pins": [34],
  "params": {
    "resolution": 12,
    "attenuation": 3
  }
}
```

#### 电压计算

```c
实际电压 = (ADC值 / 4095.0) * 3.3V
```

#### 接线注意

- **分压电路**：测量>3.3V电压必须使用电阻分压
- **滤波电容**：并联100nF电容减少噪声
- **阻抗匹配**：信号源阻抗<10kΩ

### 3.12 DAC模拟输出 (type=16/27)

#### 适用场景

- 音频输出
- 电压控制
- 波形生成
- 模拟信号模拟

#### 引脚配置

```
单引脚：25 或 26
ESP32仅GPIO25和GPIO26支持DAC
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **defaultValue** | 整数 | 0 | 0~255 | 默认输出值（8位） |

#### 电压计算

```
输出电压 = (DAC值 / 255.0) * 3.3V
```

| DAC值 | 输出电压 | 百分比 |
|-------|---------|--------|
| 0 | 0V | 0% |
| 128 | 1.65V | 50% |
| 255 | 3.3V | 100% |

#### 配置示例

```json
{
  "id": "dac_audio",
  "name": "音频DAC",
  "type": 27,
  "enabled": true,
  "pins": [25],
  "params": {
    "defaultValue": 128
  }
}
```

#### 接线注意

- **负载阻抗**：建议>10kΩ，低阻抗需加运放缓冲
- **滤波**：输出端加低通滤波器平滑波形
- **ESP32-S3**：❌ 不支持DAC功能

### 3.13 TM1637数码管 (type=47)

#### 适用场景

- 时钟显示
- 温度显示
- 计数器显示
- 简单数字显示

#### 引脚配置

```
顺序：引脚1=CLK(时钟), 引脚2=DIO(数据)
```

#### 参数说明

| 参数 | 类型 | 默认值 | 取值范围 | 说明 |
|------|------|--------|---------|------|
| **brightness** | 整数 | 3 | 0~7 | 显示亮度 |

#### 亮度级别

| 亮度值 | 相对亮度 | 适用场景 |
|--------|---------|---------|
| 0 | 最暗(1/16) | 夜间低光环境 |
| 3 | 中等(4/16) | **默认**，室内使用 |
| 7 | 最亮(16/16) | 强光环境 |

#### 配置示例

```json
{
  "id": "display_clock",
```

#### 接线注意

- **5V供电**：TM1637支持3.3V和5V，5V更亮
- **电平兼容**：ESP32 3.3V信号可直接驱动
- **线长**：建议<30cm，长距离需加电容

## 四、引脚分配规则

### 4.1 ESP32引脚限制

#### ❌ 不能使用的引脚

| 引脚 | 原因 |
|------|------|
| GPIO6~11 | 连接Flash芯片 |
| GPIO34~39 | 仅输入，无内部上拉 |

#### ⚠️ 需谨慎使用的引脚

| 引脚 | 注意事项 |
|------|---------|
| GPIO0 | 启动时为输入，拉低进入下载模式 |
| GPIO15 | 启动时日志输出，可能短暂高电平 |
| GPIO34~39 | 只能做输入，不能输出 |

#### ✅ 推荐使用的引脚

| 用途 | 推荐引脚 |
|------|---------|
| 数字输出 | 2, 4, 5, 12, 13, 14, 15, 16, 17, 18, 19, 21, 22, 23, 25, 26, 27, 32, 33 |
| 数字输入 | 所有GPIO（除6~11） |
| PWM输出 | 与数字输出相同 |
| ADC输入 | 32, 33, 34, 35, 36, 39 |
| I2C | 21(SDA), 22(SCL) |
| SPI | 18(SCK), 19(MISO), 23(MOSI), 5(CS) |
| 触摸 | 0, 2, 4, 12, 13, 14, 15, 27, 32, 33 |

---

### 4.2 引脚冲突检测

#### 冲突场景

| 场景 | 问题 | 解决方案 |
|------|------|---------|
| 两个外设使用同一引脚 | 后初始化的覆盖先前的 | 修改其中一个的引脚 |
| 引脚被系统占用 | 初始化失败 | 查阅引脚分配表 |
| I2C/SPI共享引脚 | 通信冲突 | 使用不同引脚组 |

#### 检查方法

1. 在外设列表中查看所有引脚配置
2. 导出 `peripherals.json` 搜索重复引脚
3. 使用引脚分配表格记录

## 六、常见错误与解决方案

### 6.1 配置错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| **外设不工作** | enabled=false | 启用外设 |
| **引脚初始化失败** | 引脚冲突 | 检查并修改引脚分配 |
| **传感器读数异常** | 引脚接错或类型错误 | 确认接线和sensorType |
| **JSON格式错误** | 手动编辑语法错误 | 使用Web界面或JSON验证工具 |
| **I2C设备不响应** | 引脚错误或地址不对 | 使用I2C扫描器确认地址 |
| **舵机抖动** | 供电不足 | 使用外部5V电源 |
| **LED亮度不够** | 未串联电阻或电阻过大 | 计算合适阻值 |

### 6.2 排查步骤

1. **检查启用状态**：确认 `enabled: true`
2. **查看状态值**：状态=4表示错误
3. **查看日志**：Web界面或串口日志
4. **验证接线**：使用万用表测量
5. **简化测试**：先测试单个外设
6. **检查冲突**：导出配置搜索重复引脚
