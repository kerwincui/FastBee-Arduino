# 示例教程

> 从零开始的典型场景配置教程。

## 示例 1：温湿度监测站

### 硬件准备

- ESP32 开发板
- DHT22 温湿度传感器
- 面包板和杜邦线

### 接线

| DHT22 引脚 | ESP32 引脚 |
|-----------|-----------|
| VCC | 3.3V |
| GND | GND |
| DATA | GPIO 4 |

### 配置步骤

1. 烧录固件并登录设备
2. 进入**外设配置** → 添加外设：
   - 类型：传感器 (37)
   - 数据引脚：4
   - 传感器型号：DHT22
   - 名称：温室传感器
3. 进入**外设执行** → 添加规则：
   - 触发器：定时 (`0 */10 * * * *`，每10分钟)
   - 动作：读取传感器 (`READ_SENSOR`)
   - 动作：上报传感器 (`REPORT_SENSOR`)

> 传感器分类和接线详情请参阅 [传感器完整指南](../peripherals/sensor-guide-complete.md)。

## 示例 2：智能灯控

### 硬件准备

- ESP32 开发板
- 5V 继电器模块
- LED 灯泡

### 接线

| 继电器引脚 | ESP32 引脚 |
|-----------|-----------|
| VCC | 5V |
| GND | GND |
| IN | GPIO 16 |

### 配置步骤

1. 进入**外设配置** → 添加外设：
   - 类型：数字输出 (12)
   - GPIO：16
   - 名称：客厅灯
2. 进入**外设执行** → 添加规则：
   - 规则名称：定时开灯
   - 触发器：定时 (`0 0 18 * * *`，每天18:00)
   - 动作：设置 GPIO (`SET_GPIO` → HIGH)
3. 再添加规则（关灯）：
   - 触发器：定时 (`0 0 23 * * *`，每天23:00)
   - 动作：设置 GPIO (`SET_GPIO` → LOW)

> 完整动作类型列表请参阅 [外设执行](../periph-exec/README.md)。

## 示例 3：MQTT 远程控制

### 前提

- 已有 MQTT 服务器（如 FastBee 平台、EMQX、Mosquitto）
- ESP32 已连接 WiFi

### 配置步骤

1. 进入**协议配置** → MQTT：
   - 服务地址：`iot.fastbee.cn`
   - 端口：1883
   - 客户端ID：`FB-LivingRoom-01`
   - 用户名/密码：（由平台提供）
2. 添加继电器（参考示例 2）
3. 进入**外设执行** → 添加规则：
   - 规则名称：MQTT 开关灯
   - 触发器：MQTT，主题 `cmd/livingroom/light`
   - 条件：payload 为 `ON` 时开灯，`OFF` 时关灯

## 示例 4：Modbus 传感器采集

### 硬件准备

- ESP32 开发板
- Modbus RTU 温湿度传感器（如 SHT20 Modbus 版）
- RS485 转 TTL 模块（MAX485）

### 接线

| MAX485 引脚 | ESP32 引脚 |
|------------|-----------|
| VCC | 5V |
| GND | GND |
| DI | GPIO 17 (TX) |
| RO | GPIO 16 (RX) |
| RE+DE | GPIO 18 |

### 配置步骤

1. 进入**协议配置** → Modbus RTU：
   - 启用 Modbus
   - 波特率：9600
   - 数据位：8，校验：无，停止位：1
2. 进入**外设配置** → 添加 Modbus 子设备：
   - 从站地址：1
   - 功能码：03（读取保持寄存器）
   - 起始地址：0
   - 寄存器数量：2
3. 进入**外设执行** → 添加规则：
   - 触发器：定时 (`0 */5 * * * *`，每5分钟)
   - 动作：读取 Modbus 寄存器 (`MODBUS_READ`)

> 引脚分配原则请参阅 [外设配置](../peripherals/README.md)。
